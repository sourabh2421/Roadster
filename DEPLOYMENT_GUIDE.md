# Roadster Booking Backend - Deployment Guide

## ✅ What We Built

A complete serverless booking backend with:
- ✅ MongoDB Atlas connection (reusable, optimized for serverless)
- ✅ 4 API endpoints (availability check, create booking, list bookings, update status)
- ✅ Server-side overlap validation (prevents double-booking)
- ✅ Input validation (phone numbers, dates, required fields)
- ✅ CORS enabled for all endpoints

### Step 3: Deploy to Vercel

```bash

```

#### Option B: Vercel Dashboard
1. Go to [vercel.com](https://vercel.com)
2. Import your GitHub repository
3. Vercel will auto-detect Vite project
4. Click "Deploy"

**Critical:** You MUST add `MONGODB_URI` to Vercel or the API will fail.

2. Navigate to: **Settings → Environment Variables**
3. Add:

## 🔍 Troubleshooting

### API returns "MONGODB_URI not found"
- **Solution:** Add `MONGODB_URI` to Vercel environment variables (Step 4)

### 409 Conflict Error on Booking
- **Cause:** Booking time overlaps with existing booking
- **Solution:** Choose a different time slot or car

### 400 Bad Request
- **Cause:** Missing or invalid fields
- **Solution:** Check API_DOCUMENTATION.md for required fields

### CORS Error in Browser
- **Cause:** Headers not being set properly
- **Solution:** All endpoints have CORS enabled - check if Vercel deployed correctly

### Connection Timeout
- **Cause:** MongoDB network access not configured
- **Solution:** Add `0.0.0.0/0` to IP whitelist in MongoDB Atlas
